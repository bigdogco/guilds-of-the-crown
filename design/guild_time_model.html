Guild — Time Model

Time unit hierarchy, fixed timestep loop, game triggers, speed control, and tick rate table

Structure

Time unit hierarchy

Seven nested time units from tick to year. Weeks per season and seasons per year are configurable in options. All other unit relationships are fixed.

Implementation

Fixed timestep accumulator

A tick is not the same as dt. The game loop receives variable dt each frame and uses an accumulator to fire a consistent number of fixed-size game ticks. Speed control changes tickDuration — not the loop rate.

Triggers

What each time unit triggers

Each time unit fires a distinct set of game events. Ticks drive everything continuously. Days and weeks are the main financial settlement points.

Timing

Tick rates and real-time durations

Based on 1 in-game day = 10 real minutes at normal speed. Fast speed is 4x normal. Slow speed is 0.25x normal. All values derive from 60 ticks per in-game minute.

Speed control

Player speed control

4 speed settings — pause, slow, normal, fast. All simulation logic scales with tick rate. Player UI and notifications are always responsive regardless of speed.

Design decisions

Configurable options

Only two values are exposed in game options to keep the settings screen simple.
